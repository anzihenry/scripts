## v0.2.0

这个版本把 `macos-scripts` 从“可安装”继续推进到“可发布、可验证、职责更清晰”的状态，重点收口了 Homebrew 安装与配置链路，并补齐 release 自动化。

### Highlights

- 新增 `release publish|verify` 命令和底层 `maintain/github_release_publish.sh`，可以用 `gh` 非交互创建或更新 GitHub Release
- `setup/homebrew-setup.sh` 已收口为“仅配置/修复已有 Homebrew”，首次安装职责正式移交给 `bootstrap/install.sh`
- `setup brew configure` 新增真正的 `--dry-run`，可做无副作用 smoke test
- 抽出并复用 `setup/lib/homebrew_config.sh`，统一 Homebrew shellenv、镜像、repo sync 和 dry-run 逻辑
- stable Formula、bootstrap 文案和 README 链路继续向正式发布口径对齐

### Included Improvements

- `macos-scripts release verify <tag>`
- `macos-scripts release publish <tag>`
- `macos-scripts setup brew configure --dry-run`
- `bootstrap/install.sh --dry-run --yes --skip-configure`
- install mode 下的日志与配置目录约定
- 更清晰的 Homebrew bootstrap / configure 职责边界

### Bootstrap

全新 macOS 可使用以下命令启动安装：

```bash
BOOTSTRAP_TAG="v0.2.0"
curl -fsSL "https://raw.githubusercontent.com/anzihenry/scripts/${BOOTSTRAP_TAG}/bootstrap/install.sh" | zsh
```

### Notes

- `release publish <tag>` 支持传入裸版本号，例如 `0.2.0`，CLI 会自动规范化为 `v0.2.0`
- `setup brew configure --dry-run` 可用于发布前 smoke test 和安装后排障
- stable Formula 将指向 `v0.2.0` 源码归档；`brew install --HEAD anzihenry/scripts/macos-scripts` 仍保留给开发版安装场景
