# v0.2.0 发布清单

本文档用于 `v0.2.0` 发布准备，覆盖版本锚点、发布前验证、tag / GitHub Release 操作，以及 stable Formula 回填步骤。

## 发布目标

- 版本号：`0.2.0`
- Git tag：`v0.2.0`
- 发布分支：`main`
- 发布定位：release 自动化 + Homebrew bootstrap / configure 职责拆分 + dry-run 完整化版本

## 发布前确认

- 工作区变更已审阅完毕，确认没有不应进入 `v0.2.0` 的临时改动。
- [bin/macos-scripts](bin/macos-scripts) 中的 `MACOS_SCRIPTS_VERSION` 为 `0.2.0`。
- [bootstrap/install.sh](bootstrap/install.sh) 中的 `BOOTSTRAP_RELEASE_TAG` 默认为 `v0.2.0`。
- [README.md](README.md) 中 bootstrap、release publish / verify 示例均使用 `v0.2.0`。
- [releases/v0.2.0-release-notes.md](releases/v0.2.0-release-notes.md) 已与本次发布内容一致。
- 仓库中不再残留面向用户的 `main` 分支 bootstrap raw URL。

## 发布前验证

在仓库根目录执行：

```bash
./bin/macos-scripts --version
./bin/macos-scripts release verify 0.2.0 || true
./bin/macos-scripts setup brew configure --dry-run
zsh bootstrap/install.sh --dry-run --yes --skip-configure
zsh lint/lint_shell.sh bootstrap/install.sh bin/macos-scripts setup maintain job lint
zsh -n bootstrap/install.sh setup/*.sh maintain/*.sh job/*.sh lint/*.sh bin/macos-scripts
bash -n lib/colors.sh setup/git_forge_ssh_setup.sh maintain/github_release_publish.sh
```

固定发布前验证项仍建议包含一次“本地真实安装实测”，用于在远端 tap 更新前验证当前工作区快照：

```bash
cat >/tmp/macos-scripts-install-test.sh <<'EOF'
#!/bin/zsh
set -e
set -u
set -o pipefail

WORKSPACE="/absolute/path/to/scripts"
TMPDIR_ROOT="$(mktemp -d)"

cleanup() {
  set +e
  if brew list --formula macos-scripts >/dev/null 2>&1; then
    HOMEBREW_NO_AUTO_UPDATE=1 brew uninstall --formula macos-scripts >/dev/null 2>&1 || true
  fi
  if brew tap | grep -qx 'anzihenry/scripts'; then
    HOMEBREW_NO_AUTO_UPDATE=1 brew untap anzihenry/scripts >/dev/null 2>&1 || true
  fi
  rm -rf "$TMPDIR_ROOT"
}
trap cleanup EXIT

mkdir -p "$TMPDIR_ROOT/work"
rsync -a --exclude '.git' --exclude '.DS_Store' "$WORKSPACE/" "$TMPDIR_ROOT/work/"
cd "$TMPDIR_ROOT/work"

git init -b main >/dev/null
git config user.name 'Test Runner'
git config user.email 'test@example.com'
git add . >/dev/null
git commit -m 'test snapshot' >/dev/null
git clone --bare "$TMPDIR_ROOT/work" "$TMPDIR_ROOT/repo.git" >/dev/null

cd "$WORKSPACE"

echo '=== stable install test ==='
HOMEBREW_NO_AUTO_UPDATE=1 brew tap --custom-remote anzihenry/scripts "file://$TMPDIR_ROOT/repo.git"
HOMEBREW_NO_AUTO_UPDATE=1 brew install anzihenry/scripts/macos-scripts
./bin/macos-scripts --version

HOMEBREW_NO_AUTO_UPDATE=1 brew uninstall --formula macos-scripts
HOMEBREW_NO_AUTO_UPDATE=1 brew untap anzihenry/scripts

echo '=== bootstrap test ==='
HOMEBREW_NO_AUTO_UPDATE=1 TAP_REMOTE_URL="file://$TMPDIR_ROOT/repo.git" zsh bootstrap/install.sh --yes --skip-configure
macos-scripts --version
EOF

chmod +x /tmp/macos-scripts-install-test.sh
zsh /tmp/macos-scripts-install-test.sh
```

验证结果应满足：

- `macos-scripts --version` 输出 `macos-scripts v0.2.0`
- bootstrap dry-run 显示的 raw URL、tag 和安装流程均为 `v0.2.0`
- `brew install anzihenry/scripts/macos-scripts` 可解析到 stable Formula
- 使用临时本地 tap 的 stable install 实测可成功安装并输出 `macos-scripts v0.2.0`
- 使用 `TAP_REMOTE_URL=file://...` 的 bootstrap 实测可成功安装并输出 `macos-scripts v0.2.0`
- 实测结束后，`brew list --formula macos-scripts` 与 `brew tap | grep '^anzihenry/scripts$'` 均不应残留测试状态
- lint 和语法检查全部通过

## Tag 与 Release 操作

确认 `main` 已经是待发布提交后执行：

```bash
git checkout main
git pull --ff-only origin main
git add bin/macos-scripts bootstrap/install.sh maintain/github_release_publish.sh README.md releases/v0.2.0-release-notes.md releases/v0.2.0-checklist.md
git commit -m "Prepare release v0.2.0"
git push origin main
git tag -a v0.2.0 -m "Release v0.2.0"
git push origin v0.2.0
```

发布后校验：

```bash
git ls-remote --tags origin | grep "refs/tags/v0.2.0$"
curl -I https://raw.githubusercontent.com/anzihenry/scripts/v0.2.0/bootstrap/install.sh
./bin/macos-scripts release verify v0.2.0
```

## GitHub Release 操作

```bash
./bin/macos-scripts release publish v0.2.0 --yes
```

底层等价命令：

```bash
bash maintain/github_release_publish.sh \
  --tag v0.2.0 \
  --notes-file releases/v0.2.0-release-notes.md \
  --update-existing \
  --yes
```

## Stable Formula 回填

由于 GitHub 源码归档 `sha256` 需要基于远端 `v0.2.0` tarball 计算，打完 tag 后再更新 [Formula/macos-scripts.rb](Formula/macos-scripts.rb)：

```bash
curl -L https://github.com/anzihenry/scripts/archive/refs/tags/v0.2.0.tar.gz -o /tmp/macos-scripts-v0.2.0.tar.gz
shasum -a 256 /tmp/macos-scripts-v0.2.0.tar.gz
```

然后把 Formula 中的 `url` / `sha256` / `version` 更新到 `v0.2.0`，再补一次提交推送到 `main`。
